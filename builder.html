If you haven't set a Proxy URL: the snippet carries your API key in client-side code — anyone viewing source can read it. Fine for testing on a single page; for production, use the Proxy setup tab to deploy a free Cloudflare Worker that keeps your key server-side.

        

      


      
      

        
A proxy is a tiny one-file server that holds your API key and forwards browser requests to Breeze. This solves CORS (the reason a raw API key won't fetch from a browser) and keeps your key invisible to site visitors. Pick a host below — both are free.


        

          
          
        


        
        

          
Vercel's deploy flow is currently the simplest paste-and-go setup. No CLI, no Git required, no greyed-out buttons. About 3 minutes start to finish.

          
1

            
Sign in to Vercel

            
Go to vercel.com/signup and sign up — easiest with GitHub, Google, or email. Free tier is plenty.

          

          
2

            
Create a new project from a template

            
From your dashboard click Add New → Project, then Browse Templates, and pick the Other category → Next.js (or any starter — we'll replace the code). Or skip templates: click Add New → Project → Continue with empty repository. Name it breeze-proxy and create it.

          

          
3

            
Add the proxy file

            
In the Vercel editor (or the GitHub repo it created), make a new file at api/v1/products.js and paste the code below. Replace YOUR_BREEZE_API_KEY_HERE with your real key.

            

          

          
4

            
Add the key as an environment variable (safer)

            
In your Vercel project: Settings → Environment Variables → add BREEZE_API_KEY with your key as the value. The code reads from process.env.BREEZE_API_KEY first and falls back to the inline string, so doing this means your key never lives in the source file.

          

          
5

            
Deploy and copy the URL

            
Click Deploy. Vercel gives you a URL like https://breeze-proxy-yourname.vercel.app. Paste that into the Proxy URL field in the left rail and you're done — preview goes live and the embed snippet no longer contains your key.

          

          

            
            
For production: uncomment the ALLOWED_ORIGINS block in the code and add your website's domain so random sites can't use your proxy to hammer the Breeze API on your bill.
